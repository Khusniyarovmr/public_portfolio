В мессенджере реализованы следующие фичи:

(1 балл) Период жизни доставленных сообщений — 1 час (по умолчанию).
(1 балл) Клиент может отправлять не более 20 (по умолчанию) сообщений в общий чат в течение определенного периода - 1 час (по умолчанию). В конце каждого периода лимит обнуляется;
(2 балла) Возможность пожаловаться на пользователя. При достижении лимита в 3 предупреждения, пользователь становится "забанен" - невозможность отправки сообщений в течение 4 часов (по умолчанию);
(3 балла) Возможность отправлять файлы различного формата (объёмом не более 5Мб, по умолчанию).
(3 балла) Пользователь может подключиться с двух и более клиентов одновременно. Состояния должны синхронизироваться между клиентами.

Если кратко:
/ban username - забанить пользователя. Надо 3 бана, чтобы получилось. Можно прописать сразу 3 раза с одного клиента.
/status - Получить статус о бане и возможности писать сообщения в общий чат.
/sendfile username filepath - отправить файл.
/getfiles - получить файлы
/exit - выйти из чата
@username - написать приватное сообщение пользователю.

===================================================================

Все сообщения записываются в БД SQLite3 в таблицу massage.

В отдельном потоке работает система, которая помечает сообщения как "удаленные", если сообщение было создано более часа назад.

Чтобы пожаловаться на пользователя надо написать следующее:
/ban username, где username - имя участника чата.
Если пользователь получает 3 бана, то он не может больше отправлять и получать сообщения на протяжении определенного времени.

Чтобы отправить пользователю приватное сообщение, надо писать следующее:
@username message, где message - это сообщение
Сообщение увидит только пользователь, которому оно было направлено.

Чтобы отправить файл, надо написать следующее:
/sendfile username filepath
Пользователь, которому отправлен файл получит сообщение о том, что ему отправили файл.
Другой пользователь, чтобы получить файл должен написать /getfiles.
Есть две папки: client_local_files и server_local_files.
При отправлении файла, файла сначала записывается в папку server_local_files в папку пользователя, которому он был направлен.
Далее, после того как другой пользователь написал /getfiles, файл будет скопирован в client_local_files в папку пользователя.
По сути будет происходить синхронизация папок на сервере и на клиенте.

Чтобы просмотреть свой статус, надо написать следующее:
/status
Если у пользователя нет никаких блокировок и граничений, то будет сообщение "All good!"
Если у пользователя действует ограничение по сообщениям или бан, то ему отобразится сообщение с указанием количества оставшегося времени.

Чтобы выйти из чата надо написать следующее:
/exit
Клиент закрывается.
